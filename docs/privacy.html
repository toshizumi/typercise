プライバシー

Typercise が何を送り、何を送らないか、どう止められるか。

送信する情報（オンの場合）

「世界の Typercizer」ランキングのため、希望者のみ 1日1回、前日分の集計値を送信します。

  • client_id — アプリが初回起動時にローカルで生成する UUID v4。Apple ID 等とは紐付きません
  • date — 集計対象日（YYYY-MM-DD、お使いの端末のローカル日付）
  • keys / corrections — その日の総打鍵数と訂正キー数
  • kcal — 換算カロリー（参考値）
  • peak_kpm / avg_kpm / active_minutes — 速度・活動時間
  • nickname — ランキング表示用に任意で設定したニックネーム（未設定時は送信しません）
  • app_version / os_version — 互換性確認用

送信しない情報

  • キーの内容（押されたキーが何だったか）
  • 入力先のアプリ、ウィンドウ、ファイル、URL
  • クリップボードの内容、選択中のテキスト
  • ホスト名、ユーザー名、メールアドレス、Apple ID
  • 位置情報、IP アドレスは Worker に届きますが サーバ側のログにも残しません

送信を止める方法

アプリ側で

メニューバーアイコン → ポップオーバーを開く → ヘッダ右上の歯車 ⚙ → 「世界の Typercizer に参加」のチェックを外す。即座に送信は停止します。

初回起動時

初回起動で出る同意モーダルの「送信せずに使う」を選ぶと、最初から送信されません。

ファイルを直接編集

UI が開けない場合: ~/Library/Application Support/jp.garage-standard.keycount/settings.json を編集して "telemetry_enabled": false にしてください。

過去のデータの削除

サーバには client_id を主キーにしたあなたの日次集計値のみがあります。完全削除を希望する場合は GitHub Issues か apple@garage-standard.jp に client_id をご連絡ください。手元の settings.json を削除すれば、次回起動から別の UUID で開始するため、過去のデータと事実上切り離せます。

第三者提供・広告

ありません。データは 世界の Typercizer ページの集計表示にのみ利用します。

検証可能性

Typercise はオープンソースです。クライアントから何が送信されるか、サーバで何が保存されるかは、すべて GitHub で読めます。

  • クライアント送信処理: src-tauri/src/telemetry.rs
  • サーバ実装: server/src/index.ts
  • サーバ DB スキーマ: server/migrations/0001_init.sql
